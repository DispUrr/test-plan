# План автоматизации тестирования формы отправки заявки на курс “Тестировщик ПО” сайта netology.ru

## Уровни тестирования
1. Модульное тестирование - проверка функциональности и поиск дефектов - переход со страницы на страницу, кликабельность кнопок;
1. Приёмочное тестирование - удовлетворяет ли система приёмочным критериям, соответствие ТЗ;

## Виды тестирования
1. Функциональное тестирование - тестирование функциональности на соответствие требованиям;
1. Конфигурационное тестирование - проверка работы при различных конфигурациях системы (разные браузеры, ПК и мобильное устройство);
1. Дымовое тестирование - поткрывается ли веб-сайт, достапна ли страница с курсом "Тестировщик ПО".

### **Ожидаемый результат:**
Ожидаемый результат проверок 1-6: форма записи на курс найдена.

### **Форма записи в блоке "Запишитесь или получите консультацию"**

Валидные данные:
* имя как минимум из двух букв латиницей или кириллицей без цифр и спецсимволов (Мария, Petr);
* номер телефона из 11 цифр, начинающийся на +7 (79513576325);
* e-mail латиницей в формате name@mail.ru, name@bk.ru, name@list.ru, name@google.com.

Невалидные данные:
* имя менее одной буквы латиницей или кириллицей, имя со спецсимволами и цифрами или только цифрами (O1ga, Va$ilij, 12345);
* телефон менее 11 символов, более 11 символов, буквы в номере (9513526, 7951357412563, 79O67531524,79l67513526);
* e-mail без указания домена.

## Перечень используемых инструментов:
* Java - язык программирования для написания автотестов, для этого языка есть удобная среда интегрированной разработки IntelliJ IDEA, программирование на этом языке было изучено в рамках курса;
* IntelliJ IDEA - интегрированная среда разработки ПО, анализирует код, указывает на ошибки, предлагает примеры решения;
* Gradle - система автоматической сборки, которая используется для работы с Java;
* JUnit5 - инфраструктура модульного тестирования для Java, современная и стабильная;
* Selenide - для автоматизированного тестирования веб-приложений на основе Selenium WebDriver, дающий такие преимущества, как изящный API, мощные селекторы, простая конфигурация;
* REST-Assured - Java-библиотека для автоматизации тестирования REST-API;
* Faker - для генерации данных (имя, телефон, e-mail);
* Allure - фреймворк для создания отчётов по тестированию.

## Перечень возможных рисков при автоматизации:
* Недоступность сайта или страницы;
* Проблема полного покрытия кода автотестами;
* Необходимость модифицировать автотесты при частом изменении страницы;
* Увеличение стоимости автоматизации тестирования;
* Отсутствие тестовых данных;
* Отсутствие доступа к реальной базе данных.
